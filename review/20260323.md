## [347. Top K Frequent Elements](../new/20260223/347_top_k_frequent_element.md)

- first intuition is to use a map to store the frequency of each value, so for e.g 1 1 1 2 2 3 => 1 - 3, 2 - 2, 3 - 1
- then how we could get first K element
- we use a 2nd dimentional array with the value we found above as index => then the values we store in those index is the key of the map that have that frequency
- one final loop go from 1 to k and append to the result list, could potentially overshoot greater than k if the input does not guarantee unique frequency
